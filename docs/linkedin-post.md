# LinkedIn Post Draft

I built a todo app, but not as a meme portfolio project.

I built it because I needed it.

A while ago, I was struggling with time management. My manager gave me very simple advice: write down where your time is going.

So I hacked together Ground: a small time and intentions tracker that fits how I think about my day.

It is not trying to be the best productivity app for everyone. That is the point.

The sections are where they make sense to me. The order of things is how my brain expects them. The categories reflect the kind of work I actually do. The live app is private because it contains my real work data, but I made the repo public because the idea behind it feels worth sharing.

I have been thinking about this as a shift in who software can be built for.

Enterprise software made software valuable for organizations.

SaaS made one product economically viable for many teams.

Cloud hosting made small apps easy to run.

And now AI-assisted development is making useful software viable for much smaller audiences: one team, one workflow, sometimes one person.

Not every useful app needs to become a startup. Not every internal tool needs to be generalized into a SaaS product. Some software can simply fit the person or team using it.

I like calling this boutique software.

Small, opinionated, personal, and built around how someone actually works.

Ground is my tiny example of that. A todo app, yes. But also a reminder that software does not always need to scale its audience to justify its existence. Sometimes it just needs to work beautifully for the person it was made for.
